Navigating the Server Browser

Each database server is laid out exactly the same way in the Server Browser. This provides a consistent experience for you to navigate through the databases. Your interaction with the Server Browser is entirely based on the current selected item. Edrych changes your context for new queries and the like entirely based on the selected node.

The Levels

The Server Browser uses four levels: Server, Database, Tables/Views, and Columns. The Database and Tables/Views are separated by a folder level, which groups Tables and Views separately.

Context Menu

You can right-click on a node in the tree to bring up the context menu. From here, you can take actions on a specific node in the tree. Currently, you can perform the following actions:

New Query

Clicking New Query on a node will open up a new query window connected to the server the node is under, using the parent database of the node (if you click on the server, it will use the default).

Refresh

Clicking the Refresh button on a node will cause it to collapse the node and refresh the children information of that node. This is greatly useful when using Edrych to alter the structures, and then seeing the results of your changes immediately.
